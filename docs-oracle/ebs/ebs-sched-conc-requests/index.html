EBS R12 — Scheduled Concurrent Requests Query

Returns active scheduled concurrent requests in EBS R12.2.10 with human-readable schedule text and last/next run metadata. Useful for auditing what is scheduled, when it last ran, and what its recurrence pattern is.

Exploring set ups of fully custom scheduled job tracker, summary view updated dynamically as new custom jobs are scheduled, errors or missed run.

Prerequisites​

Access Required
  • Read access to APPS schema tables: FND_CONCURRENT_REQUESTS, FND_CONC_RELEASE_CLASSES, FND_CONCURRENT_PROGRAMS_TL, FND_USER, FND_LOOKUP_VALUES
  • Update the NAME_PREFIX% filter constant before running (see Filtering)

Query​

Show Full SQL
WITH
  base AS (
      SELECT
          r.request_id                   AS requestId,
          p.user_concurrent_program_name AS programName,
          r.description,
          s.user_name                    AS requester,
          c.class_type                   AS classType,
          c.class_info                   AS classInfo, 
          NVL(r.actual_completion_date, r.last_update_date) AS lastRunDate,   
          r.requested_start_date                            AS nextRunDate,  
          flvStatus.meaning                                 AS lastRunStatus
      FROM
          apps.fnd_concurrent_requests r
          JOIN apps.fnd_conc_release_classes c
            ON c.application_id  = r.release_class_app_id
           AND c.release_class_id = r.release_class_id
          JOIN apps.fnd_concurrent_programs_tl p
            ON p.concurrent_program_id = r.concurrent_program_id
          JOIN apps.fnd_user s
            ON r.requested_by = s.user_id
          LEFT JOIN apps.fnd_lookup_values flvStatus
            ON flvStatus.lookup_type      = 'CP_STATUS_CODE'
           AND flvStatus.lookup_code      = r.status_code 
           AND NVL(flvStatus.enabled_flag,'Y') = 'Y' 
           AND flvStatus.start_date_active IS NOT NULL 
      WHERE
          r.phase_code    = 'P'
          AND NVL(c.date2, SYSDATE + 1) > SYSDATE
          AND c.class_type IS NOT NULL
          AND UPPER(NVL(r.description, p.user_concurrent_program_name)) LIKE 'NAME_PREFIX%'
  ),
  dates AS (
      SELECT
          b.requestId AS requestId,
          LISTAGG(TO_CHAR(lvl), ', ') WITHIN GROUP (ORDER BY lvl) AS dates
      FROM
          base b
          CROSS JOIN (SELECT LEVEL AS lvl FROM dual CONNECT BY LEVEL <= 31)
      WHERE
          b.classType            = 'S'
          AND SUBSTR(b.classInfo, lvl, 1) = '1'
      GROUP BY b.requestId
  )
SELECT
    b.requestId,
    NVL2(b.description,
        b.description || ' (' || b.programName || ')',
        b.programName
    )                                                          AS concProg,
    b.requester,
    lastRunDate,   
    nextRunDate,  
    lastRunStatus,
    DECODE(b.classType,
        'P', 'Periodic',
        'S', 'On Specific Days',
        'X', 'Advanced',
        b.classType
    )                                                          AS scheduleType,
    CASE
        WHEN b.classType = 'P' THEN
            'Repeat every '
            || SUBSTR(b.classInfo, 1, INSTR(b.classInfo, ':') - 1)
            || DECODE(SUBSTR(b.classInfo, INSTR(b.classInfo,':', 1, 1) + 1, 1),
                'N', ' minutes', 'H', ' hours', 'D', ' days', 'M', ' months')
            || DECODE(SUBSTR(b.classInfo, INSTR(b.classInfo,':', 1, 2) + 1, 1),
                'S', ' from the start of the prior run',
                'C', ' from the completion of the prior run')
        WHEN b.classType = 'S' THEN
            NVL2(d.dates, 'Dates: ' || d.dates, NULL)
            || DECODE(SUBSTR(b.classInfo, 32, 1), '1', 'Last day of month ')
            || NVL(
                DECODE(SUBSTR(b.classInfo, 33, 7),
                    '1111111', 'Daily',
                    '0111110', 'Every weekday',
                    NULL),
                DECODE(SIGN(INSTR(SUBSTR(b.classInfo, 33, 7), '1')), 1,
                    'Days of week: '
                    || DECODE(SUBSTR(b.classInfo, 33, 1), '1', 'Su ')
                    || DECODE(SUBSTR(b.classInfo, 34, 1), '1', 'Mo ')
                    || DECODE(SUBSTR(b.classInfo, 35, 1), '1', 'Tu ')
                    || DECODE(SUBSTR(b.classInfo, 36, 1), '1', 'We ')
                    || DECODE(SUBSTR(b.classInfo, 37, 1), '1', 'Th ')
                    || DECODE(SUBSTR(b.classInfo, 38, 1), '1', 'Fr ')
                    || DECODE(SUBSTR(b.classInfo, 39, 1), '1', 'Sa ')
                )
            )
            || DECODE(SUBSTR(b.classInfo, 40, 5),
                '00000', NULL, '11111', NULL, NULL, NULL,
                'Weeks: '
                || DECODE(SUBSTR(b.classInfo, 40, 1), '1', '1st ')
                || DECODE(SUBSTR(b.classInfo, 41, 1), '1', '2nd ')
                || DECODE(SUBSTR(b.classInfo, 42, 1), '1', '3rd ')
                || DECODE(SUBSTR(b.classInfo, 43, 1), '1', '4th ')
                || DECODE(SUBSTR(b.classInfo, 44, 1), '1', '5th ')
            )
            || NVL(
                DECODE(SUBSTR(b.classInfo, 45, 12), '111111111111', ', Every Month'),
                DECODE(SIGN(INSTR(SUBSTR(b.classInfo, 45, 12), '1')), 1,
                    'in ' || TRIM(BOTH ' ' FROM
                        DECODE(SUBSTR(b.classInfo, 45, 1), '1', 'Jan ', '')
                        || DECODE(SUBSTR(b.classInfo, 46, 1), '1', 'Feb ', '')
                        || DECODE(SUBSTR(b.classInfo, 47, 1), '1', 'Mar ', '')
                        || DECODE(SUBSTR(b.classInfo, 48, 1), '1', 'Apr ', '')
                        || DECODE(SUBSTR(b.classInfo, 49, 1), '1', 'May ', '')
                        || DECODE(SUBSTR(b.classInfo, 50, 1), '1', 'Jun ', '')
                        || DECODE(SUBSTR(b.classInfo, 51, 1), '1', 'Jul ', '')
                        || DECODE(SUBSTR(b.classInfo, 52, 1), '1', 'Aug ', '')
                        || DECODE(SUBSTR(b.classInfo, 53, 1), '1', 'Sep ', '')
                        || DECODE(SUBSTR(b.classInfo, 54, 1), '1', 'Oct ', '')
                        || DECODE(SUBSTR(b.classInfo, 55, 1), '1', 'Nov ', '')
                        || DECODE(SUBSTR(b.classInfo, 56, 1), '1', 'Dec ', '')
                    ) || '.',
                ', Every Month')
            )
    END AS schedule
FROM
    base b
    LEFT JOIN dates d ON d.requestId = b.requestId
ORDER BY b.description;

Filtering​

Update Before Running

Change NAME_PREFIX% in the WHERE clause to match your program name prefix. Use '%' to return all scheduled requests with no name filter.

FilterPurpose
r.phase_code = 'P'Pending (scheduled in manager), not already running or completed
NVL(c.date2, SYSDATE + 1) > SYSDATERelease class is still active/valid
c.class_type IS NOT NULLRequest has a defined schedule
UPPER(NVL(...)) LIKE 'NAME_PREFIX%'Scope to your custom programs; falls back to program name if description is null

classInfo Reference​

classInfo is a VARCHAR2 bit string stored by EBS to encode schedule patterns. It is always processed as character data — never cast to a number.

Bit PositionLengthMeaningExample
1–3131Specific days of month1 at position 5 = 5th of month
321Last day of month1 = enabled
33–397Days of week Su Mo Tu We Th Fr Sa0111110 = every weekday
40–445Week of month 1st–5th10100 = 1st and 3rd week
45–5612Months Jan–Dec111111111111 = every month
Common Pitfall: "First day shows as 16"

This happens when code numerically casts the bit string — leading zeros are dropped, shifting all positions. This query uses SUBSTR(...) = '1' and INSTR(..., '1') on character data exclusively, so positions are always accurate.

Schedule Decoding​

  • Periodic (P)
  • Specific Days (S)

classInfo format: interval:unit:mode

TokenValuesMeaning
intervalNumberHow often
unitN H D MMinutes, Hours, Days, Months
modeS CFrom start of prior run / From completion of prior run

Output example: Repeat every 4 hours from the completion of the prior run

Assembled in this order — each piece is optional and concatenated:

  1. Explicit dates — if any of bits 1–31 are set → Dates: 1, 5, 20
  2. Last day — if bit 32 set → Last day of month
  3. Day of week (shortcutted first):
    • 1111111 → Daily
    • 0111110 → Every weekday
    • Any other pattern → Days of week: Mo Tu Fr
  4. Week of month — only if some but not all of bits 40–44 are set → Weeks: 1st 3rd; omitted if 00000 (all weeks) or 11111 (all weeks)
  5. Months — if bits 45–56 = 111111111111 → , Every Month; if partial → in Jan Mar; if none → defaults to , Every Month

Output example: Days of week: Mo Tu We Th Fr, Every Month

Deep Dive — Performance & Extensibility

Performance Notes​

  • The base CTE applies all WHERE filters early, minimizing rows passed to dates and the final SELECT.
  • dates uses a single CONNECT BY LEVEL <= 31 cross join with LISTAGG — no row-by-row processing or PL/SQL.
  • All schedule decoding uses SUBSTR/INSTR/DECODE on character data — no type conversions or scalar subqueries.
  • FND_LOOKUP_VALUES is joined once with enablement and date filters inline, not as a subquery per row.

Extensibility​

Locale-aware day/month names: Replace hard-coded tokens ('Mo ', 'Jan ') with a small lookup CTE joined by position index and LISTAGG-ed.

Duplicate status meanings: If FND_LOOKUP_VALUES returns multiple rows for a status_code (e.g., entries like @Normal alongside Normal), wrap the join target in a subquery using ROW_NUMBER() OVER (PARTITION BY lookup_code ORDER BY ...) to select one canonical meaning per code. Alternatively, add AND flvStatus.meaning NOT LIKE '@%' as a quick filter if the @ prefix pattern is consistent in your instance.
